# Bugs

### Bug-01: API позволяет создавать объявление с отрицательной стоимостью
Краткое описание: API принимает отрицательное значение поля "price" и успешно создает объявление, 
что противоречит бизнес логике.

#### Шаги воспроизведения:

1)Отправить POST-запрос на "https://qa-internship.avito.com/api/1/item"

2)Передать тело в запрос:
{
  "sellerID": 445677,
  "name": "Iphone 12 pro max 256GB",
  "price": -1000,
  "statistics": {
    "contacts": 2414368,
    "likes": 1,
    "viewCount": 1
  }
}

#### Фактическое поведение:

Сервис успешно создает объявление, возвращает 200 "OK" и сообщение с успешным созданием объявления и его "id"

#### Ожидаемое поведение:

Сервер должен вернуть 400 Bad Request и сообщение об ошибке валидации

#### Серьезность:

Major - стоимость товара не может быть <= 0 у.е. и также такие значения могут ломать сортировку товаров по стоимости

### Bug-02: API не позволяет создавать объявления со стоимостью 0 рублей
Краткое описание: API не позволяет создавать объявления, стоимость которых 0 рублей,
люди могут отдавать вещи бесплатно.

#### Шаги воспроизведения:

1)Отправить POST-запрос на "https://qa-internship.avito.com/api/1/item"

2)Передать тело в запрос:
{
  "sellerID": 678567,
  "name": "Пылесос REDMOND",
  "price": 0,
  "statistics": {
  "contacts": 49876589,
  "likes": 9,
  "viewCount": 1
  }
}

3)Получить от сервера

#### Фактическое поведение:

Сервис не создает объявление со стоимостью 0 рублей, хотя это не противоречит логике и возвращает 400 Bad Request

#### Ожидаемое поведение:

Сервер должен принимать цену 0 как валидное значение, возвращать 200 OK и успешно создавать объявление с присвоенным id

#### Серьезность:

Minor - блокирует сценарий, при котором пользователь хочет отдать свою вещь за бесплатно, но не основной функционал.
Можем указать стоимость 1 рубль, тем самым обойдя этот баг

### Bug-03: API позволяет создавать объявления с отрицательными значениями полей statistics

#### Краткое описание:

При создании объявления через API можно передать отрицательные значения в поля объекта 
statistics (contacts, likes, viewCount). Сервер принимает такие данные и возвращает успешный ответ 200 OK, что нарушает 
логическую целостность данных.

#### Шаги воспроизведения:

1)Отправить POST-запрос на "https://qa-internship.avito.com/api/1/item"

2)Передать тело в запрос:
  {
  "sellerID": 678567,
  "name": "Пылесос REDMOND",
  "price": 0,
  "statistics": {
    "contacts": -1234,
    "likes": -92,
    "viewCount": -1
    }
  }

3)Получить ответ от сервера

#### Фактическое поведение:

Сервер возвращает статус код 200 OK и успешно создаёт объявление с отрицательными значениями в полях статистики

#### Ожидаемое поведение:

Сервер должен возвращать ошибку валидации 400 Bad Request с сообщением о том, что поля статистики не могут принимать 
отрицательные значения

#### Серьезность:

Major - нарушает целостность данных, может привести к некорректной работе аналитики, сортировки и ранжирования 
объявлений. Поля likes/viewCount/contacts — это счётчики, они не могут быть < 0 по смыслу.

### Bug-04: API позволяет создавать объявления с отрицательным полем sellerId

#### Краткое описание:
При создании объявления через API система принимает отрицательное значение поля sellerId и успешно 
сохраняет объект, игнорируя бизнес-валидацию идентификатора продавца

#### Шаги воспроизведения:

1)Отправить POST-запрос на "https://qa-internship.avito.com/api/1/item"

2)Передать тело в запрос:
  {
  "sellerID": -1,
  "name": "phone 11 pro 512GB",
  "price": 9990,
  "statistics": {
    "contacts": 2344,
    "likes": 1,
    "viewCount": 18
    }
  }

3)Получить ответ от сервера

#### Фактическое поведение:

Сервер возвращает статус код 200 OK и сообщение "Сохранили объявление". Объявление успешно создаётся с невалидным 
идентификатором продавца sellerId: -1

#### Ожидаемое поведение:

Сервер должен отклонять запрос с ошибкой валидации 400 Bad Request и сообщать, что поле sellerId должно содержать 
положительное целое число

#### Серьезность:

Major - обход валидации ключевого идентификатора. Может привести к проблемам с привязкой объявлений к продавцам, некорректной 
работе фильтрации/поиска и нарушению целостности данных в БД.

### Bug-05: API возвращает успешный ответ при запросе списка объявлений по невалидному sellerId

#### Краткое описание: 

При выполнении GET-запроса к эндпоинту /api/1/{sellerId}/item с невалидными значениями sellerId (0 или отрицательное 
число), сервер возвращает статус 200 OK вместо ожидаемой ошибки валидации 400 Bad Request

#### Шаги воспроизведения:

1)Отправить POST-запрос на "https://qa-internship.avito.com/api/1/-1/item"

Вместо -1 можем поставить любое отрицательное число или 0

2)Проверить код ответа и тело ответа

#### Фактическое поведение:

Сервер возвращает статус код 200 OK и пустой массив [] (или список объявлений). Ошибка валидации не генерируется, хотя 
идентификатор продавца не может быть нулевым или отрицательным.

#### Ожидаемое поведение:

Сервер должен отклонять запрос с ошибкой валидации 400 Bad Request и сообщением о некорректном формате идентификатора 
продавца, так как sellerId должен быть положительным целым числом

#### Серьезность:

Minor - cистема не падает, возвращает безопасный ответ, не приводит к потере или порче данных
