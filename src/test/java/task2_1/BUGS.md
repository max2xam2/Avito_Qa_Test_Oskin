# Bugs

### Bug-01: API позволяет создавать объявление с отрицательной стоимостью
Краткое описание: API принимает отрицательное значение поля "price" и успешно создает объявление, 
что противоречит бизнес логике.
Шаги воспроизведения:
1)Отправить POST-запрос на "https://qa-internship.avito.com/api/1/item"

2)Передать тело в запрос:
{
  "sellerID": 445677,
  "name": "Iphone 12 pro max 256GB",
  "price": -1000,
  "statistics": {
    "contacts": 2414368,
    "likes": 1,
    "viewCount": 1
  }
}

Фактическое поведение

Сервис успешно создает объявление, возвращает 200 "OK" и сообщение с успешным созданием объявления и его "id"

Ожидаемое поведение

Сервер должен вернуть 400 Bad Request и сообщение об ошибке валидации

Серьезность

Скриншоты

### Bug-02: API не позволяет создавать объявления со стоимостью 0 рублей
Краткое описание: API не позволяет создавать объявления, стоимость которых 0 рублей,
люди могут отдавать вещи бесплатно.

Шаги воспроизведения:
1)Отправить POST-запрос на "https://qa-internship.avito.com/api/1/item"

2)Передать тело в запрос:
{
  "sellerID": 678567,
  "name": "Пылесос REDMOND",
  "price": 0,
  "statistics": {
  "contacts": 49876589,
  "likes": 9,
  "viewCount": 1
  }
}

Фактическое поведение

Сервис не создает объявление с стоимостью 0 рублей, хотя это не противоречит
логике. И возвращает 400 Bad Request

Ожидаемое поведение

Сервер должен был вернуть 200 OK и присвоить id объявлению

Серьезность

Скриншоты
